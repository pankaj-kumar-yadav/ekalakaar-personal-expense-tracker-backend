## 1. Project Overview

| Field | Detail |
|---|---|
| **Project** | Personal Expense Tracker |
| **Stack** | MERN (MongoDB, Express, React, Node.js) |
| **Backend role** | REST API to create, list, and delete expenses stored in MongoDB |
| **Frontend contract** | React app consumes these APIs for add, view, delete, and total-spent display |

### Objective

Build a simple full-stack Expense Tracker that allows users to **add**, **view**, and **delete** expenses. The backend must expose three REST endpoints and persist data in MongoDB.

---

## 2. Backend Requirements (Assignment)

### Required REST APIs

| Method | Endpoint | Purpose |
|---|---|---|
| `POST` | `/api/expenses` | Add a new expense |
| `GET` | `/api/expenses` | Get all expenses |
| `DELETE` | `/api/expenses/:id` | Delete an expense by ID |

### Data Storage

- Use **MongoDB** to store expense documents.
- Each expense must support the fields required by the frontend form:
  - **Amount**
  - **Description**
  - **Category**
  - **Date**

### Out of Scope (Not Required by Assignment)

- User authentication / authorization
- Update/edit expense (`PUT` / `PATCH`)
- Pagination, filtering, or search
- File uploads or receipts
- Rate limiting, caching, or advanced analytics

## 4. Expense Data Model

### MongoDB Document Schema

```ts
{
  _id: ObjectId,        // auto-generated by MongoDB
  amount: number,       // required, positive number
  description: string,  // required
  category: string,     // required (e.g. Food, Transport, Bills)
  date: Date,           // required — expense date
  createdAt: Date,      // optional — auto timestamp
  updatedAt: Date       // optional — auto timestamp
}
```

### Validation Rules

| Field | Rules |
|---|---|
| `amount` | Required; must be a number > 0 |
| `description` | Required; non-empty string |
| `category` | Required; non-empty string |
| `date` | Required; valid ISO date string or Date object |

## 5. API Specification

Base URL (local): `http://localhost:8080`

All expense routes are mounted under `/api`.

---

### 5.1 POST `/api/expenses` — Add Expense

**Request body (JSON):**

```json
{
  "amount": 250,
  "description": "Lunch at cafe",
  "category": "Food",
  "date": "2026-07-14"
}
```

**Success response:** `201 Created`

```json
{
  "success": true,
  "data": {
    "_id": "64abc123...",
    "amount": 250,
    "description": "Lunch at cafe",
    "category": "Food",
    "date": "2026-07-14T00:00:00.000Z",
    "createdAt": "2026-07-14T10:30:00.000Z",
    "updatedAt": "2026-07-14T10:30:00.000Z"
  }
}
```

**Error responses:**

| Status | When |
|---|---|
| `400` | Missing or invalid fields |
| `500` | Server or database error |

---

### 5.2 GET `/api/expenses` — Get All Expenses

**Success response:** `200 OK`

```json
{
  "success": true,
  "count": 2,
  "totalAmount": 750,
  "data": [
    {
      "_id": "64abc123...",
      "amount": 250,
      "description": "Lunch at cafe",
      "category": "Food",
      "date": "2026-07-14T00:00:00.000Z"
    },
    {
      "_id": "64abc456...",
      "amount": 500,
      "description": "Bus pass",
      "category": "Transport",
      "date": "2026-07-13T00:00:00.000Z"
    }
  ]
}
```

> **Note:** `totalAmount` is computed on the backend (sum of all `amount` values). The frontend assignment requires showing total spent — this field avoids extra client-side logic but is optional if the frontend sums locally.

**Sorting (recommended):** Newest first (`date` descending, or `createdAt` descending).

**Error responses:**

| Status | When |
|---|---|
| `500` | Server or database error |

---

### 5.3 DELETE `/api/expenses/:id` — Delete Expense

**URL param:** `id` — MongoDB ObjectId of the expense.

**Success response:** `200 OK`

```json
{
  "success": true,
  "message": "Expense deleted successfully",
  "data": {
    "_id": "64abc123...",
    "amount": 250,
    "description": "Lunch at cafe",
    "category": "Food",
    "date": "2026-07-14T00:00:00.000Z"
  }
}
```

**Error responses:**

| Status | When |
|---|---|
| `400` | Invalid MongoDB ObjectId format |
| `404` | Expense not found |
| `500` | Server or database error |

## 10. Frontend Integration Notes

The React frontend will call these endpoints. Ensure:

| Concern | Backend action |
|---|---|
| CORS | `cors()` enabled (already in `app.ts`) |
| JSON | `express.json()` middleware (already in `app.ts`) |
| Date format | Accept ISO strings (`"2026-07-14"`); store as `Date` |
| ID for delete | Return `_id` in POST/GET responses so frontend can pass it to DELETE |
| Total spent | Either return `totalAmount` in GET response or let frontend sum `amount` |
