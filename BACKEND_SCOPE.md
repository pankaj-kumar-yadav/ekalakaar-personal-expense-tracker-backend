## 1. Project Overview

| Field | Detail |
|---|---|
| **Project** | Personal Expense Tracker |
| **Stack** | MERN (MongoDB, Express, React, Node.js) |
| **Backend role** | REST API to create, list, and delete expenses stored in MongoDB |
| **Frontend contract** | React app consumes these APIs for add, view, delete, and total-spent display |

### Objective

Build a simple full-stack Expense Tracker that allows users to **add**, **view**, and **delete** expenses. The backend must expose three REST endpoints and persist data in MongoDB.

## 2. Backend Requirements (Assignment)

### Required REST APIs

| Method | Endpoint | Purpose |
|---|---|---|
| `POST` | `/api/expenses` | Add a new expense |
| `GET` | `/api/expenses` | Get all expenses |
| `DELETE` | `/api/expenses/:id` | Delete an expense by ID |

### Data Storage

- Use **MongoDB** to store expense documents.
- Each expense must support the fields required by the frontend form:
  - **Amount**
  - **Description**
  - **Category**
  - **Date**

### Out of Scope (Not Required by Assignment)

- User authentication / authorization
- Update/edit expense (`PUT` / `PATCH`)
- Pagination, filtering, or search
- File uploads or receipts
- Rate limiting, caching, or advanced analytics

## 3.1 Coding Conventions ‚Äî Const & Enum (Required)

Use **enums** and **named consts heavily**. Avoid scattering magic strings/numbers across controllers, schemas, and helpers.

| Prefer | Avoid |
|---|---|
| `Period.THIS_WEEK` | `"this-week"` inline |
| `ActivityRange.WEEK` | `"week"` inline |
| `ExpenseCategory.FOOD` | `"Food"` inline |
| `DASHBOARD.ACTIVITY_LIMIT` | bare `8` |
| `DASHBOARD.KPI_LABELS.TOTAL_SPENT` | `"Total Spent"` inline |

**Rules:**

1. Put shared enums/consts under `src/constants/` (e.g. `dashboard.ts`, `expense.ts`).
2. Use TypeScript `enum` for closed string sets (query periods, activity ranges, expense categories).
3. Use `as const` objects for related defaults, limits, and response labels.
4. Zod schemas must validate against the same enums (`z.enum(Period)`, `z.enum(ExpenseCategory)`, etc.) ‚Äî not duplicated string arrays.
5. Controllers, helpers, seed data, and route defaults must import and use these symbols; do not re-declare the literal values.

**Current shared modules:**

| Module | Contents |
|---|---|
| `src/constants/dashboard.ts` | `Period`, `ActivityRange`, `DASHBOARD` (defaults, KPI labels, activity limit) |
| `src/constants/expense.ts` | `ExpenseCategory` (`Food`, `Transport`, `Bills`, `Other`) |

## 4. Expense Data Model

### MongoDB Document Schema

```ts
{
  _id: ObjectId,        // auto-generated by MongoDB
  amount: number,       // required, positive number
  description: string,  // required
  category: ExpenseCategory, // required ‚Äî Food | Transport | Bills | Other
  date: Date,           // required ‚Äî expense date
  createdAt: Date,      // optional ‚Äî auto timestamp
  updatedAt: Date       // optional ‚Äî auto timestamp
}
```

### Validation Rules

| Field | Rules |
|---|---|
| `amount` | Required; must be a number > 0 |
| `description` | Required; non-empty string |
| `category` | Required; must be one of `ExpenseCategory` enum values |
| `date` | Required; valid ISO date string or Date object |

## 5. API Specification

Base URL (local): `http://localhost:8080`

All expense routes are mounted under `/api`.

---

### 5.1 POST `/api/expenses` ‚Äî Add Expense

**Request body (JSON):**

```json
{
  "amount": 250,
  "description": "Lunch at cafe",
  "category": "Food",
  "date": "2026-07-14"
}
```

**Success response:** `201 Created`

```json
{
  "success": true,
  "data": {
    "_id": "64abc123...",
    "amount": 250,
    "description": "Lunch at cafe",
    "category": "Food",
    "date": "2026-07-14T00:00:00.000Z",
    "createdAt": "2026-07-14T10:30:00.000Z",
    "updatedAt": "2026-07-14T10:30:00.000Z"
  }
}
```

**Error responses:**

| Status | When |
|---|---|
| `400` | Missing or invalid fields |
| `500` | Server or database error |

---

### 5.2 GET `/api/expenses` ‚Äî Get All Expenses

**Success response:** `200 OK`

```json
{
  "success": true,
  "count": 2,
  "totalAmount": 750,
  "data": [
    {
      "_id": "64abc123...",
      "amount": 250,
      "description": "Lunch at cafe",
      "category": "Food",
      "date": "2026-07-14T00:00:00.000Z"
    },
    {
      "_id": "64abc456...",
      "amount": 500,
      "description": "Bus pass",
      "category": "Transport",
      "date": "2026-07-13T00:00:00.000Z"
    }
  ]
}
```

> **Note:** `totalAmount` is computed on the backend (sum of all `amount` values). The frontend assignment requires showing total spent ‚Äî this field avoids extra client-side logic but is optional if the frontend sums locally.

**Sorting (recommended):** Newest first (`date` descending, or `createdAt` descending).

**Error responses:**

| Status | When |
|---|---|
| `500` | Server or database error |

---

### 5.3 DELETE `/api/expenses/:id` ‚Äî Delete Expense

**URL param:** `id` ‚Äî MongoDB ObjectId of the expense.

**Success response:** `200 OK`

```json
{
  "success": true,
  "message": "Expense deleted successfully",
  "data": {
    "_id": "64abc123...",
    "amount": 250,
    "description": "Lunch at cafe",
    "category": "Food",
    "date": "2026-07-14T00:00:00.000Z"
  }
}
```

**Error responses:**

| Status | When |
|---|---|
| `400` | Invalid MongoDB ObjectId format |
| `404` | Expense not found |
| `500` | Server or database error |

## 10. Frontend Integration Notes

The React frontend will call these endpoints. Ensure:

| Concern | Backend action |
|---|---|
| CORS | `cors()` enabled (already in `app.ts`) |
| JSON | `express.json()` middleware (already in `app.ts`) |
| Date format | Accept ISO strings (`"2026-07-14"`); store as `Date` |
| ID for delete | Return `_id` in POST/GET responses so frontend can pass it to DELETE |
| Total spent | Either return `totalAmount` in GET response or let frontend sum `amount` |
